# Ecosystem Add-ons

The toolkit works on its own. These add-ons can make it stronger when they fit the project.

## Useful Skills Or Agent Workflows

- Frontend/design review skills for visual hierarchy, typography, layout, accessibility, and motion.
- Planning/spec skills for turning rough ideas into implementation-ready plans.
- Skill-creation skills for maintaining this toolkit's custom workflows.
- Code-review skills for correctness, tests, security, and maintainability.
- Learning-capture skills for promoting reusable lessons back into the toolkit.

Treat these as optional. Do not add them unless they support the current workflow.

## Useful MCPs Or Product Integrations

- Notion: readable blueprint, playbook, reusable learnings log, and changelog.
- GitHub: source control, issues, PRs, review, and public distribution.
- Sentry: error reporting before beta or production usage.
- PostHog or Amplitude: product analytics and funnel measurement.
- Vercel: web deployment, preview deployments, and environment separation.

## Selection Rule

Add integrations when they remove real workflow friction. Avoid adding tools just because the
toolkit mentions them.
